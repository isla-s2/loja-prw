### Trabalho 3 - 3 de dezembro

1. Página de cadastro de usuário: deve permitir o cadastro em banco de dados
(nome, login/usuário, senha, CPF, data de nascimento e e-mail) e exibir
mensagens de sucesso ou de erro, conforme o caso – a senha deve ser
armazenada em hash;
2. Página de login com usuário e senha: deve ser possível validar o usuário e senha
cadastrado previamente no banco de dados, também deve informar se houve
sucesso ou falha no momento do login (depois de logado, deve-se criar uma
sessão no PHP);
3. Página protegida: uma página que só é acessível depois de efetuar o login (não
deve ser possível acessá-la sem estar logado) e que seja possível efetuar o logout
do sistema, sugere-se utilizar o carrinho de compras como página protegida, ele
não precisa ser funcional;
4. Página inicial do site exibindo produtos, pelo menos 6, que estão cadastrados no
banco de dados contendo foto, nome e preço.
Deve ser utilizado como base o sistema implementado previamente.


Todas as páginas criadas devem possuir HTML, CSS, JavaScripts e PHP criados pela
própria dupla. <br>
Entrega:
1. Arquivos do site (imagens, HTML, CSS, JS e PHP);
2. script de export do database no mariaDB;
3. link para vídeo postado no YouTube gravado pelos(as) próprios(as) autores(as)
demonstrando como ficou o database no mariaDB (via phpMyAdmin), o
funcionamento de cada uma das 4 páginas solicitadas e, explicando, parcialmente,
o código-fonte de cada uma delas.
